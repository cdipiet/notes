pipenv 🔗

pipenv è come npm per Node.js, gestisce pip e virtualenv
salva i virtualenv del progetto in:

  • Linux: ~/.local/share/virtualenvs
  • Windows: %USERPROFILE%/.virtualenvs

Dà automaticamente il nome al virtualenv utilizzando il nome della directory radice del progetto.

virtualenv 🔗

pipenv crea automaticamente i virtualenv, alla prima esecuzione di pipenv install <libreria> pipenv crea il Pipfile ed un virtualenv che ha come nome il nome della directory root del progetto seguito dall'hash del full path della directory root del progetto. Esiste dunque un mapping tra virtualenv creato automaticamente e path assoluto del progetto. Se si sposta il progetto nel file system bisogna cancellare e ricreare il virtualenv

Creare i virtualenv all'interno del progetto stesso 🔗

You might want to set export PIPENV_VENV_IN_PROJECT=1 in your .bashrc/.zshrc (or any shell configuration file) for creating the virtualenv inside your project’s directory, avoiding problems with subsequent path changes.

Pipfile 🔗

  • https://pipenv.pypa.io/en/latest/basics/#general-recommendations-version-control

Pipfile uses the TOML Spec

Il Pipfile viene creato automaticamente alla prima installazione di una libreria.

Esempio di Pipfile presente nella root del progetto:

[[source]]
url = "https://pypi.org/simple"
verify_ssl = true
name = "pypi"

[packages]
apscheduler = "*"
async-dns = "*"
cryptography = "*"
fastapi = "*"
requests ="*"
sqlalchemy = "*"
uvicorn = {extras = [ "standard",],version = "*"}
redis = "*"

[dev-packages]

[requires]
python_version = "3.9"

Installazione delle librerie da Pipfile 🔗

Le librerie vengono installate nel virtualenv dal Pipfile con:

# chiamare install senza argomenti installa tutti i pacchetti del Pipfile
$ pipenv install
# Crea il Pipfile.lock

Le librerie vengono installate nel virtualenv dal Pipfile.lock con:

$ pipenv install --ignore-pipfile

Per produzione 🔗

# Per produzione
$ pipenv check # cerca vulnerabilita di sicurezza
$ pipenv lock  # crea il file Pipenv.lock in formato pipenv
$ pipenv install --ignore-pipfile # usa il file lock per installare i pacchetti
$ pipenv install --dev # installa solo i pacchetti dev del pipfile

Aggiungere librerie al progetto 🔗

Esempi di aggiunta di nuove librerie al progetto, questi comandi installano le librerie nel virtualenv ed aggiornano il Pipfile

$ cd myproject
$ pipenv install # inizializza il virtualenv e lo attiva
$ pipenv install flask==0.12.1
$ pipenv install numpy
$ pipenv install pytest --dev

Aggiornare tutte le librerie del progetto 🔗

$ pipenv update --outdated  # Aggiorna le librerie vecchie
$ pipenv update             # Aggiorna tutto

Vedere le versioni attuali delle librerie installate 🔗

$ pipenv graph
$ pipenv graph --reverse
$ pipenv lock -r
$ pipenv run pip freeze

Esecuzione del progetto 🔗

Esempio di esecuzione del progetto, l'entry point dell'applicazione è main.py:

$ pipenv run python main.py

Aprire una shell attivando il virtualenv:

$ pipenv shell

Generare il Pipfile dai requirements pip 🔗

$ pipenv install -r path/to/requirements.txt

Generare i requirements in formato pip 🔗

# Per generare i requirements in formato pip:
$ pipenv lock -r > requirements.txt
$ pipenv lock -r --dev-only > dev-requirements.txt

Virtualenv 🔗

$ pipenv --venv
$ pipenv --rm # Remove project virtualenv (inferred from current directory)
$ pipenv --where
$ pipenv --py

.env 🔗
