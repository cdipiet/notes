Spostamento della cache-dir ðŸ”—

https://python-poetry.org/docs/configuration/

Config Directory

Linux: $XDG_CONFIG_HOME/pypoetry or ~/.config/pypoetry
Windows: %APPDATA%\pypoetry

Cache Directory

Linux: $XDG_CACHE_HOME/pypoetry or ~/.cache/pypoetry
Windows: %LOCALAPPDATA%\pypoetry

C:\> poetry config --list
C:\> mkdir D:\pypoetry
C:\> poetry config cache-dir D:\pypoetry
REM     ==> crea il file %APPDATA%\pypoetry\config.toml

Mancanza dei certificati delle CA in Windows ðŸ”—

https://stackoverflow.com/questions/69814872/i-receive-ssl-certificate-verify-failed-when-doing-poetry-install https://github.com/python-poetry/poetry/issues/745 SSL Error (certificate verify failed) for Poetry install

https://install.python-poetry.org in windows genera l'errore:

```python
urllib.error.URLError: <urlopen error [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed: unable to get local issuer certificate (_ssl.c:1010)>
```

https://install.python-poetry.org utilizza requests che a sua volta utilizza certifi, pacchetto che contiene i certificati delle CA.

I certificati della CA possono dunque essere installati con:

Soluzione:

C:\> pip install requests

proxy ðŸ”—

https://github.com/python-poetry/poetry/issues/3748 Using Poetry behind a company proxy

Per utilizzare il proxy con poetry e pycharm:

- impostata la variabile di ambiente windows HTTPS_PROXY : http://<user>:<pass>@<server>:<port>
- configurato il proxy in pycharm

```bash
    $ export HTTP_PROXY="http://<user>:<pass>@<server>:<port>"
    $ export HTTPS_PROXY="http://<user>:<pass>@<server>:<port>"
```
