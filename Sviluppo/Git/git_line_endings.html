Voglio normalizzare un repository in modo che i file di testo utilizzino solamente \n

Configurazione globale:

git config -l --show-origin
git config --global core.autocrlf false
git config --global core.eol lf
git config --show-origin --get core.autocrlf
git ls-files --eol

When a text file is normalized, its line endings are converted to LF in the repository. Setting the text attribute on a path enables end-of-line normalization and marks the path as a text file.
End-of-line conversion takes place without guessing the content type.

Creazione di un file .gitattributes di progetto:

git add . -A
git commit -m "Saving files before refreshing line endings"
git add --renormalize .
git status
git commit -m "Normalize all the line endings"
