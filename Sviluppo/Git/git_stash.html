  2. Se ci sono modifiche locali e vogliamo annullarle

    # Se ci sono modifiche locali e vogliamo annullarle

    # Se ci sono modifiche locali e vogliamo mantenerle

Mentre stiamo lavorando sul main ci viene chiesto di apportare una fix urgente, normalmente salveremmo il lavoro attuale in un branch temporaneo, ritornando sul branch main per apportare la fix:

# Salviamo il lavoro corrente:
# ... work work work ... Arriva una richiesta di fix urgente, il ramo main ha index e working dir modificati
$ git switch -c my_wip      # si crea un nuovo ramo e ci si sposta, index e working directory vengono mantenuti
$ git commit -a -m "WIP"    # salviamo con un nuovo commit il lavoro attuale, il ramo main ora Ã¨ pulito

# torniamo sul main, ora pulito, per lavorare alla fix:
$ git switch main           
$ edit emergency fix
$ git commit -a -m "Fix in a hurry" # commit per la fix sul main

# Si ripristina il lavoro precedente:
$ git switch my_wip
$ git reset --soft HEAD^ # Elimina l'ultimo commit ripristinando lo stato di index e working directory
# ... continue working ...
# Al termine del lavoro si riporta tutto sul main:
$ git commit -a -m "WIP"
$ git switch main
$ git merge my_wip
$ git branch -d my_wip
# Oppure:
$ git rebase main

$ git stash pop
# ... continue working ...
