Caricamento del progetto locale nel repository github - push -u ðŸ”—

  1. Creare un progetto vuoto su github, chiamandolo ad esempio Notes, impostarlo come Privato:
    https://github.com/cdipiet/Notes Pagina Home del nuovo progetto

  2. Dalla pagina principale del progetto github premere il pulsante "SSH" in modo da avere le URL SSH

  3. Copiare la url SSH, quindi non https ma git@: git@github.com:cdipiet/Notes.git

  4. Creare il progetto git locale se giÃ  non esistente:

    $ echo "# Notes" >> README.md
$ git init
$ git add . # oppure git add README.md
# Adds the files in the local repository and stages them for commit. To unstage a file, use 'git reset HEAD YOUR-FILE'.
$ git commit -m "First commit"
# Commits the tracked changes and prepares them to be pushed to a remote repository. To remove this commit and modify the file, use 'git reset --soft HEAD~1' and commit and add the file again.

  5. Agganciare il repository locale al progetto github e caricare i files

    Attenzione ad utilizzare SSH e non HTTPS

    $ git remote add origin git@github.com:cdipiet/Notes.git
# Sets the new remote
$ git remote -v
# Verifies the new remote URL
$ git push -u origin master
# Pushes the changes in your local repository up to the remote repository you specified as the origin

Note that master stands for master branch of your repository and origin is automatically created reference to the original repository.

Per sovrascrivere un branch remoto con i file locali.

  1. Prima bisogna creare il repository in GitHub: https://github.com/new
  2. Poi da https://docs.github.com/en/github/administering-a-repository/setting-the-default-branch impostare il default branch a main se giÃ  non lo Ã¨.
  3. Creare un repository locale (git init) e fare il push --force
#!/usr/bin/env bash
# git-reset.sh
# Sovrascrittura del repository remoto con i file locali, utilizzando "main" come nome del branch principale al posto di "master": https://github.com/github/renaming
REPO="git@github.com:cdipiet/Notes.git"
SCRIPT_PATH="${BASH_SOURCE}"
while [ -L "${SCRIPT_PATH}" ]; do
  SCRIPT_DIR="$(cd -P "$(dirname "${SCRIPT_PATH}")" >/dev/null 2>&1 && pwd)"
  SCRIPT_PATH="$(readlink "${SCRIPT_PATH}")"
  [[ ${SCRIPT_PATH} != /* ]] && SCRIPT_PATH="${SCRIPT_DIR}/${SCRIPT_PATH}"
done
SCRIPT_PATH="$(readlink -f "${SCRIPT_PATH}")"
SCRIPT_DIR="$(cd -P "$(dirname -- "${SCRIPT_PATH}")" >/dev/null 2>&1 && pwd)"
UPDIR=$(dirname $SCRIPT_DIR)
echo "SCRIPT_PATH=${SCRIPT_PATH}"
echo "SCRIPT_DIR=${SCRIPT_DIR}"
# cd $UPDIR
cd $SCRIPT_DIR
# Remove the history from
rm -rf .git
# recreate the repos from the current content only
git init
git add .
#    git add -A # stages All
#    git add .  # stages new and modified, without deleted
#    git add -u # stages modified and deleted, without new
git commit -m "Initial commit"
# appena creato il repository, il branch di default di chiama 'master'
# github consiglia di rinominare il branch di default 'master' ad es. in 'main'
# 'master' non si deve piu usare come nome del branch principale, lo rinominiamo in 'main'
#  creiamo il branch 'main' localmente, copiando la history dal master.
# git config --global init.defaultBranch main
CURRENT_BRANCH=$(git rev-parse --abbrev-ref HEAD)
[[ "$CURRENT_BRANCH" != "main" ]] && git branch -m "$CURRENT_BRANCH" main
# definisce il repository remoto
git remote add origin "$REPO"
# invia il nuovo branch 'main' al repository remoto impostandolo come default(upstream)
# init e primo commit creano un nuovo ramo/commit incompatibile con quello del server, nel senso che non ha antenati comuni.
# Per forzare la sovrascrittura nel nuovo ramo sul server serve --force
git push -u --force origin main # -u, --set-upstream
# Se il branch di default era 'master' allora HEAD sta ancora puntando al master, per rimuovere il branch master
# bisogna sbloccarlo spostando HEAD al nuovo branch 'main'(se HEAD punta ad un certo branch quel branch non puÃ² essere rimosso)
git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main
# Questo da errore se non si imposta prima il branch di default a 'main' da github "Settings --> Branches"
# git push origin --delete master
git branch -a # -a, --all List both remote-tracking branches and local branches.
# master is now named main
# If you have a local clone, you can update it by running:
# git branch -m master main
# git fetch origin
# git branch -u origin/main main
# git remote set-head origin -a

https://github.com/settings/tokens/ Personal access tokens\Tokens (classic)

Bisogna poi utilizzare tale token come password dalla commandline, il nome utente rimane lo stesso. Per generare il token Ã¨ sufficiente dare il solo ruolo repo

$ git clone https://github.com/cdipiet/Py_Language.git

https://docs.github.com/en/get-started/getting-started-with-git/about-remote-repositories#cloning-with-https-urls
