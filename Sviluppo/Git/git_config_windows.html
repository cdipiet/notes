  1. Installare Git e Notepad++

     rem aggiunta di git al path
 C:\> runas /u:administrator "C:\Windows\system32\rundll32.exe sysdm.cpl,EditEnvironmentVariables"
 rem oppure come utente semplice:
 C:\> C:\Windows\system32\rundll32.exe sysdm.cpl,EditEnvironmentVariables
 C:\> set PATH=%PROGRAMFILES%\Git\bin;%PATH%

  2. Configurazione Utente Git

    %USERPROFILE%\.gitconfig

    C:\> git config -l --show-origin
C:\> git config --show-origin core.autocrlf

rem configurazione utente
C:\> git config --global user.name Nome Cognome
C:\> git config --global user.email nome.cognome@xx.it
C:\> git config --global core.autocrlf false
C:\> git config --global init.defaultBranch main
C:\> git config --global core.editor "'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin"
C:\> git config --global log.date iso
C:\> git config --global format.pretty format:%h%x09%C(cyan)%ad%x09%Cgreen%s
C:\> REM git config --global http.proxy "http://userdomain\$USER:xxxxx@proxy.server.it:8080"
C:\> git config --global alias.amend "commit --amend --no-edit"

rem verifica dell'editor
C:\> git config --global -e
rem verifica della configurazione
C:\> type %USERPROFILE%\.gitconfig
[user]
    name = Nome Cognome
    email = nome.cognome@xx.it
[core]
    editor = 'C:/Program Files/Notepad++/notepad++.exe' -multiInst -notabbar -nosession -noPlugin
    autocrlf = false
[log]
   date = iso
   [format]
   pretty = format:"%h%x09%C(cyan)%ad%x09%Cgreen%s"

  3. Generare una coppia di chiavi privata/pubblica

    %USERPROFILE%\.ssh\id_rsa.pub %USERPROFILE%\.ssh\id_ed25519.pub

    C:\> ssh-keygen -t rsa -b 4096 -C "nome.cognome@xx.it"
rem oppure(consigliata):
C:\> ssh-keygen -t ed25519 -C "nome.cognome@xx.it"
rem copia la chiave pubblica nella clipboard
C:\> clip < %USERPROFILE%\.ssh\id_rsa.pub
rem oppure(consigliata):
C:\> clip < %USERPROFILE%\.ssh\id_ed25519.pub

  4. Aggiunta della chiave pubblica al proprio profilo Gitlab/Github

    • https://github.com/settings/keys
    • https://gitlab/profile/keys
    • https://gitlab/help/ssh/README#generating-a-new-ssh-key-pair

    /Settings/SSH Keys/Add an SSH key poi incollare con CTRL+V

  5. Verifica della connessione:

    C:\> ssh -T git@gitlab.it

  6. Scaricamento del repository:

C:\> git clone git@gitlab/myrepo.git

Aggiornare il token di accesso in windows ðŸ”—

https://stackoverflow.com/questions/70283510/how-to-update-a-github-access-token-via-command-line

In windows, dopo aver rigenerato un token in https://github.com/settings/tokens/, bisogna cancellare il token esistente:

C:\> control.exe keymgr.dll

Oppure cercare dalla ricerca di windows, Gestione Credenziali (credential manager), ed in Credenziali Windows cancellare tutte le voci relative a git.
