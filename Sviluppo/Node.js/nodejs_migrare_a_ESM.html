__dirname e __filename esistono solo in CommonJS ðŸ”—

E' possibile ridefinirle in ESM

However, in most cases, this is better:

And many Node.js APIs accept URL directly, so you can just do this:

La import dinamica Ã¨ supportata sia nei moduli CommonJS che nei moduli ESM.
Un modulo ESM puÃ² importare normalmente un modulo CommonJS. Un modulo CommonJS puÃ² importare un modulo ESM utilizzando la import dinamica inserita in una funzione async(non puÃ² farlo come istruzione al di fuori di una funzione).

  red: "red",
  blue: "blue"
};


// use_colors.cjs, just works!
const { red } = require("colors"); // colors is CJS
module.exports = {
  logRed: () => console.log(red)
};


// DOES NOT ACTUALLY WORK!!! Non Ã¨ possibile utilizzare await al livello piÃ¹ esterno del modulo
const { red } = await import("colors"); // colors is now ESM
module.exports = {
  logRed: () => console.log(red)
};

module.exports = {
  // Breaking API change: logRed is now async.
  logRed: async () => {
    // Dynamic import() works in function awaits or promises
    // NOTE: Could memoize the `import()` to cache after 1st call.
    const { red } = await import("colors");
    console.log(red);
  }
};
