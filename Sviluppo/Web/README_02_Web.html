CORS: security.fileuri.strict_origin_policy = false ðŸ”—

  • https://stackoverflow.com/questions/25504851/how-to-enable-cors-on-firefox

Se una pagina html carica delle risorse tramite Javascript, il server che contiene le risorse deve rispondere alle richieste effettuate da Javascript con degli Header CORS:

Access-Control-Allow-Origin: *

contenenti il permesso di utilizzare le risorse tramite Javascript da siti diversi da quello nel quale si trovano le risorse. Quando si lavora in locale si puÃ² desiderare che una pagina HTML che contiene Javascript che carica risorse da file system possa farlo, infatti in tal caso non c'e' un server che risponde con degli header CORS concedendo i permessi di accesso alla pagina html, per farlo:

about:config
security.fileuri.strict_origin_policy = false

Firefox Font fails to load from local filesystem Downloadable font: download failed ðŸ”—

Firefox non carica font da locale quindi il font puÃ² non essere visibile quando di carica la pagina da locale. Bisogna impostare about:config

security.fileuri.strict_origin_policy = false

Per far funzionare correttamente i fonts nelle pagine html in locale, essi debbono trovarsi in una sottodirectory della pagina html che li importa per un problema di sicurezza di firefox.

  • https://bugzilla.mozilla.org/show_bug.cgi?id=760436

Using CSS to load a font from the local disk. The structure of the testcase ( linked in this bug ) is:

./subdir/index.html -> uses ../style.css 
./style.css -> links to ./font/fonts.css 
./font/fonts.css -> defines the @font-face { 
  font-family: 'Varela';
  font-style: normal;
  font-weight: normal;
  src: local('Varela'), url('Varela-Regular.woff') format('woff');
}

Notes:

  • it works if you move things to a server
  • it also works if you make all the files in the same root level dir. The placement of the index.html in the ./subdir seems to cause the above behavior.

this is working as designed. the issue here is that for a page loaded from a file:// URI, only files in (or below) the same directory of the filesystem are considered to be "same origin", and so putting the font in a different subtree (../font/) means it will be blocked by security policy restrictions. You can relax this by setting security.fileuri.strict_origin_policy to false in about:config, but as this gives the page access to your entire local filesystem, it's something to be used with caution.

Dist targets ðŸ”—

Popper.js is currently shipped with 3 targets in mind: UMD, ESM, and ESNext. No idea what am I talking about? You are looking for UMD probably.

  • UMD Universal Module Definition: AMD, RequireJS, and globals;
  • ESM ES Modules: For webpack/Rollup or browser supporting the spec;
  • ESNext Available in /dist, can be used with webpack and babel-preset-env;

Make sure to use the right one for your needs. If you want to import it with a <script> tag, use UMD.
