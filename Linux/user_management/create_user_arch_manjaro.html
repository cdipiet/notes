To create an user without a password, use passwd -d $username after the user is created to make the password empty.

$ pacman -S sudo
$ sudo useradd -m -s /bin/bash -G wheel username
$ passwd -d username # abilita l'utente ad effettuare login senza password

    wheel will give you access to sudo
    The wheel group is a special user group used on some Unix systems,
    mostly BSD systems, to control access to the su or sudo command,
    which allows a user to masquerade as another user (usually the superuser).
    Debian-like operating systems create a group called sudo with a purpose similar to that of a wheel group.\

    Administration group, commonly used to give privileges to perform administrative actions. It has full read access to journal files and the right to administer printers in CUPS.
    Can also be used to give access to the sudo and su utilities (neither uses it by default).

  • https://wiki.archlinux.org/title/sudo

To allow members of group wheel sudo access:

    %wheel      ALL=(ALL:ALL) ALL

Tip: When creating new administrators, it is often desirable to enable sudo access for the wheel group and add the user to it,
since by default Polkit treats the members of the wheel group as administrators.
If the user is not a member of wheel, software using Polkit may ask to authenticate using the root password instead of the user password.
