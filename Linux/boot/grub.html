Using Ubuntu livecd üîó

Here assuming the Ubuntu partition is sda7,and /boot partition is sda6 (if you have a separate /boot partition). Boot up ubuntu from the livecd,open terminal and run:

    sudo -i
    mount /dev/sda7 /mnt
    mount /dev/sda6 /mnt/boot #skip this one if not have a separate /boot partition
    grub-install --root-directory=/mnt/ /dev/sda
    # If you miss ‚Äúgrub.cfg‚Äù file,use following to recreate:
        mount --bind /proc /mnt/proc
        mount --bind /dev /mnt/dev
        mount --bind /sys /mnt/sys
        chroot /mnt update-grub
        umount /mnt/sys
        umount /mnt/dev
        umount /mnt/proc
        exit

Using the cd/usb boot up with grub üîó

Boot up the cd/usb,press c in grub menu. Type:

    grub> find /boot/grub/core.img
    grub> root (hdx,y) (previous command will output the x,y)
    grub> kernel /boot/grub/core.img
    grub> boot
    # After the boot command,you‚Äôll go into grub2 menu.Select to boot up ubuntu,and run this command to restore grub:
    sudo grub-install /dev/sda

Fix Bug setparams üîó

Problema üîó

Dopo l'update grub2 windows partiva ma prima dava l'errore "error: no argument specified"

Soluzione üîó

vi /etc/grub.d/30_otheros

set root=(hd0,msdos2)

the hd0,2 format avoid the setparams command generation bug

set root=(hd0,2)

error: no argument specified

the newest grub2(1.99) requires root having an additonal argument, as:

set root='(/dev/sda,msdos1)'

I had it set the old way as

set root='(hd0,1)'

That explained the argument error.

Cambiare il sistema operativo di default üîó

sudo gedit /etc/default/grub

then edit the line:

GRUB_DEFAULT=0

to

GRUB_DEFAULT=4

save the file, then run:

sudo update-grub
