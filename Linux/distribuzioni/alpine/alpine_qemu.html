Queste configurazioni di rete non sono permanenti e vengono perdute al reboot, utilizzare NetworkManager o systemd per renderle permanenti.

Nell'host creeremo un bridge sul quale imposteremo le configurazioni di rete del NIC fisico che Ã¨ in grado di accedere ad internet. Poi sposteremo il NIC fisico e le interfacce virtuali TAP create(una per ogni VM che creeremo) all'interno del bridge.

Non essendoci VLAN ed essendo tutte le macchine nella stessa subnet, ogni macchina vedrÃ  tutte le altre.

              |
              | physical link
              |
              o br0
             / \
            /   \
    enp5s0 o     o tap0
                 |
     - - - - - - - - - - - - - - - Host / VM boundary
                 |
                 o eth0 (inside VM)

Inserendo un NIC fisico in un bridge assieme a delle altre interfacce virtuali come ad es. tap, bisogna annotarsi ip e gateway del NIC fisico. All'inserimento del NIC fisico nel bridge, l'ip del NIC fisico deve essere rimosso ed i suoi ip e gateway devono essere impostati nel bridge, in questo modo la connessione ad internet dell'host non verrÃ  perduta.

$ # nmcli conn modify Cavo connection.autoconnect no  # Si disabilita la connessione NetworkManager del NIC fisico
$ nmcli con down Cavo
$ sudo ip link add br0 type bridge                  # Creazione del bridge
$ sudo ip link set dev br0 up                       # il bridge deve essere attivato per potervi aggiungere dei nodi
$ sudo ip tuntap add dev tap0 mode tap user $USER   # creazione dell'interfaccia virtuale tap(livello 2)
$ sudo ip tuntap add dev tap1 mode tap user $USER   # 
$ sudo ip tuntap add dev tap2 mode tap user $USER   # 
$ sudo ip tuntap add dev tap3 mode tap user $USER   #

$ sudo ip link set dev tap1 up                      # 
$ sudo ip link set dev tap2 up                      # 
$ sudo ip link set dev tap3 up                      # 
$ # sudo ip link set dev enp5s0 up 
$ sudo ip link set dev tap0 master br0              # Aggiunta dei nodi al bridge, NIC fisico e TAP(per la connessione ad una VM QEMU)
$ sudo ip link set dev tap1 master br0
$ sudo ip link set dev tap2 master br0
$ sudo ip link set dev tap3 master br0
$ sudo ip link set dev enp5s0 master br0            # Spostando il NIC fisico nel bridge vengono rimossi i sui ip e routing, viene perduta la connessione ad internet
# Per non perdere la connessione ad internet la configurazione del NIC fisico enp5s0 viene spostata sul bridge
$ # sudo dhclient -v br0                            # si usa il dhcp per assegnare un IP ed il routing al bridge
                                                    # oppure si possono assegnare manualmente ip e route al default gateway:
$ sudo ip addr flush dev br0 # rimozione dell'IP dal NIC fisico
$ sudo ip addr flush dev enp5s0 
$ sudo ip route flush dev br0
$ sudo ip addr add 192.168.1.14/24 dev br0

$ bridge link show

# By default only the port VID (PVID) is allowed to ingress through any bridge port.
# the Egress Untagged flag causes packets going out of the bridge port with VID of PVID to be untagged at egress.
# The PVID flag indicates that VLAN 1 is the PVID VLAN, which means that any untagged packet coming through this bridge port gets tagged with VID 1.
# pvid     the vlan specified is to be considered a PVID at ingress.  Any untagged frames will be assigned to this VLAN.
# untagged the vlan specified is to be treated as untagged on egress.
$ sudo bridge vlan show dev tap0
$ sudo bridge vlan add vid 1 dev enp5s0 pvid untagged
$ sudo bridge vlan del vid 2 dev enp5s0
$ sudo bridge vlan add vid 1 dev tap0 pvid untagged
$ sudo bridge vlan del vid 2 dev tap0
$ sudo bridge vlan add vid 2 dev tap1 pvid untagged
$ sudo bridge vlan del vid 1 dev tap1
$ sudo bridge vlan add vid 1 dev tap2 pvid untagged
$ sudo bridge vlan del vid 2 dev tap2
$ sudo bridge vlan add vid 2 dev tap3 untagged
$ sudo bridge vlan del vid 1 dev tap3
# Note that the bridge's self implicit port is still using VID 1,
# so assigning an IP directly on the bridge as is done on some settings will now fail to communicate properly.
# If such configuration is really needed, you can set the bridge's self port in VLAN 5 too,
# with a slightly different syntax (self) because it's the bridge itself:
$ bridge vlan del dev br0 vid 1 self
$ bridge vlan add dev br0 vid 1 pvid untagged self
$ sudo bridge vlan show

Host - Creazione della VM QEMU di base Alpine_0 ðŸ”—

La configurazione di partenza di QEMU vuole che venga specificato un MAC Address(altrimenti viene impostato quello di default generando conflitti se si utilizza piÃ¹ di una VM), i MAC Address di QEMU devono iniziare per 52:54:00, eccome come generare uno:

printf -v macaddr "52:54:00x:%02x:%02x:%02x" $(( $RANDOM & 0xff )) $(( $RANDOM & 0xff)) $(( $RANDOM & 0xff ))
echo "$macaddr"

# Host
# Installazione del SO
# -boot order: c (first hard disk), d (first CD-ROM), 
$ qemu-system-x86_64 -machine type=q35,accel=kvm -m 2G -cpu host -smp 2 -vga virtio -object rng-random,id=rng0,filename=/dev/urandom -device virtio-rng-pci,rng=rng0 -nic tap,ifname=tap0,script=no,downscript=no,model=virtio-net-pci,mac=52:54:00:21:34:56 -boot order=d -cdrom alpine-virt-3.15.0-x86_64.iso -hda ./Alpine_0.qcow2
# login come root, nessuna password
