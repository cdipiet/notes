sysctl in Arch/Manjaro 🔗

In Manjaro non si utilizza più il file /etc/sysctl.conf.

/etc/sysctl.conf ==> /etc/sysctl.d/99-sysctl.conf

sysctl è un tool che serve ad esaminare ed impostare i parametri del kernel a runtime, è implementato in procfs per mezzo del file system virtuale /proc/

Ordine di esecuzione dei file di configurazione sysctl:

$ man sysctl.d
    /etc/sysctl.d/*.conf
    /run/sysctl.d/*.conf
    /usr/lib/sysctl.d/*.conf

Files in /etc/ are reserved for the local administrator, who may use this logic to override the configuration files installed by vendor packages. It is recommended to prefix all filenames with a two-digit number and a dash, to simplify the ordering of the files.
Files in /etc/ override files with the same name in /run/, /usr/local/lib/, and /usr/lib/.
All configuration files are sorted by their filename in lexicographic order, regardless of which of the directories they reside in.
If multiple files specify the same option, the entry in the file with the lexicographically latest name will take precedence.
Thus, the configuration in a certain file may either be replaced completely (by placing a file with the same name in a directory with higher priority), or individual settings might be changed (by specifying additional settings in a file with a different name that is ordered later).

  • systemd carica le impostazioni da /etc/sysctl.d/*.conf e /usr/lib/sysctl.d/*.conf con precedenza su /etc per file omonimi
  • i file vengono eseguiti in ordine lessicografico di nome, il nome deve terminare in .conf ed iniziare con due cifre per fornire l'ordine di esecuzione
  • se un parametro è presente in piu files vince il file eseguito per ultimo(esecuzione in ordine lessicografico).
  • se due file hanno lo stesso nome vince quello in /etc/sysctl.d considerata directory a disposizione degli amministratori che vogliono effettuare override sulle configurazioni sysctl salvate in /usr/lib/sysctl.d dalle installazioni dei pacchetti. Per esempio parametri omonimi contenuti in /usr/lib/sysctl.d/50-default.conf saranno sovrascritti dagli omonimi in /etc/sysctl.d/50-default.conf

From version 207 and 21x, systemd only applies settings from /etc/sysctl.d/*.conf and /usr/lib/sysctl.d/*.conf.
If you had customized /etc/sysctl.conf,
you need to rename it as /etc/sysctl.d/99-sysctl.conf.
If you had e.g. /etc/sysctl.d/foo, you need to rename it to /etc/sysctl.d/foo.conf. The sysctl preload/configuration file can be created at /etc/sysctl.d/99-sysctl.conf.

systemd-analyze 🔗

Permette di vedere in quale file di configurazione un parametro è stato definito

$ systemd-analyze cat-config sysctl.d
$ systemd-analyze cat-config sysctl.d | grep net.ipv4.ip_forward
$ systemd-analyze cat-config sysctl.d | grep fs.inotify.max_user_watches

sysrq 🔗

  • SYSRQ

I comandi:

sysctl kernel.sysrq=1 o echo "1" > /proc/sys/kernel/sysrq

non cambiano in maniera persistente l'impostazione del parametro, sono temporanei ed al reboot vengono perduti.

$ locate sysctl.d
$ cd /usr/lib/sysctl.d/
$ grep sysrq /usr/lib/sysctl.d/*
/usr/lib/sysctl.d/50-default.conf:# Use kernel.sysrq = 1 to allow all keys.
/usr/lib/sysctl.d/50-default.conf:# See https://www.kernel.org/doc/html/latest/admin-guide/sysrq.html for a list
/usr/lib/sysctl.d/50-default.conf:kernel.sysrq = 16
$ systemd-analyze cat-config sysctl.d --no-pager # per vedere la configurazione attuale
$ sudo sysctl --all| grep rq
kernel.sysrq = 16

$ sudo bash -c "cat > /etc/sysctl.d/60-sysrq.conf" << 'EOF'
kernel.sysrq = 1
EOF

$ sysctl --system # carica tutti i file di configurazione
$ sysctl --all    # visualizza tutti i parametri attuali in /proc/sys/kernel
$ sudo sysctl --all| grep rq

Incrementare il numero di file controllabili da inotify 🔗

Utile ad esempio per PyCharm ed AndroidStudio

  • https://youtrack.jetbrains.com/articles/IDEA-A-2/Inotify-Watches-Limit
$ systemd-analyze cat-config sysctl.d | grep fs.inotify.max_user_watches
    /usr/lib/sysctl.d/10-manjaro.conf
$ man sysctl
$ man sysctl.d

$ sysctl -n fs.inotify.max_user_watches
$ sudo sysctl -a|grep fs.inotify.max_user_watches
$ sudo bash -c "cat > /etc/sysctl.d/70-max_user_watches.conf" << 'EOF'
fs.inotify.max_user_watches = 524288
EOF
sudo sysctl -p --system # ricaricare la configurazione

Diminuire lo swappiness 🔗

I valori più vicini allo zero usano più RAM prima di passare allo swap.

Aggiungere vm.swappiness=10 alla fine del file /etc/sysctl.d/99-sysctl.conf

$ sudo bash -c "cat >> /etc/sysctl.d/99-sysctl.conf" << 'EOF'
vm.swappiness=10
EOF

$ sudo sysctl vm.swappiness=10; cat /proc/sys/vm/swappiness
$ sudo sysctl --system
$ sudo sysctl -a

Aumentare il numero di file descriptor disponibili 🔗

for increasing the limit on Red Hat 5.X systems:

$ sudo sysctl -a --pattern ^fs.

$ sudo bash -c "cat >> /etc/sysctl.d/99-sysctl.conf" << 'EOF'
fs.file-max=70000
EOF


$ # echo “fs.file-max=70000″ >> /etc/sysctl.d/99-sysctl.conf
sysctl -p
$ echo “* hard nofile 65536″ >> /etc/security/limits.conf
$ echo “session required pam_limits.so” >> /etc/pam.d/login
$ ulimit -n -H
65536
