Find ðŸ”—

  • https://man.archlinux.org/man/find.1

-exec command

Execute command; true if 0 status is returned. All following arguments to find are taken to be arguments to the command until an argument consisting of ; is encountered. The string {} is replaced by the current file name being processed.

  • The specified command is run once for each matched file.
  • The command is executed in the starting directory.
  • {} ; might need to be escaped (with a \) or quoted to protect them from expansion by the shell.

-exec command {} +

the command line is built by appending each selected file name at the end; the total number of invocations of the command will be much less than the number of matched files.

Esempio di exec ðŸ”—

Vogliamo copiare e rinominare i files che si trovano in una directory senza scendere nelle sottodirectory,
e potendo processare i nomi dei file trovati prima della copia.

Prima soluzione, con find ðŸ”—

find /tmp/source -maxdepth 1 -type f -exec bash -c '
  cp $0 /tmp/dest/$(basename ${0%.src})
' {} \;

il file corrente {} viene passato a exec bash che per la sintassi di -c viene passato come parametro posizionale $0

Seconda soluzione, con ls ðŸ”—

Equivalente con ls:

for f in $(ls -1Ap /tmp/source | grep -v '/$'); do
  cp /tmp/source/$f /tmp/dest/$(basename ${f%.src})
done
  • -1 divide la lista dei files con \n
  • -A non produce . e ..
  • -p appende / ai nomi delle directory

exec ed il terminatore ; ðŸ”—

Il ; fa parte della sintassi dell'opzione exec, tuttavia essendo il ; anche un carattere speciale della shell, al fine di evitare che venga 'consumato' dalla shell durante il processo di parsing, e farlo arrivare "intero" al comando find-exec, bisogna quotarlo: \;

-exec command ;

Execute command; true if 0 status is returned. All following arguments to find are taken to be arguments to the command until an argument consisting of ';' is encountered. The string '{}' is replaced by the current file name being processed everywhere it occurs in the arguments to the command, not just in arguments where it is alone, as in some versions of find. Both of these constructions might need to be escaped (with a '\') or quoted to protect them from expansion by the shell. See the EXAMPLES section for examples of the use of the '-exec' option. The specified command is run once for each matched file. The command is executed in the starting directory. There are unavoidable security problems surrounding use of the -exec option; you should use the -execdir option instead.

Lavorare con nomi di file che contengono spazi ðŸ”—

Bisogna sostituire il normale separatore dei nomi di file in elenco spazio con il carattere ASCII 0.

$ find . -print0 | while read -d $'\0' file
do
  echo -v "$file"
done

Dollar Single Quoted String ðŸ”—

La sintassi del dollaro $ davanti alla stringa single quoted '\0' serve ad abilitare l'interpretazione dei codici escape(https://stackoverflow.com/questions/11966312/how-does-the-leading-dollar-sign-affect-single-quotes-in-bash).

Altro modo con IFS:

find . -print0 | while IFS= read -r -d '' file; do echo "$file"; done
find . -print0 | while IFS= read -r -d $'\0' file; do echo "$file"; done
# IFS= permette di trattare nomi di file contenenti spazi tab e newline, impostando IFS alla stringa nulla si evita il trim degli spazi 
# -r    impedisce l'interpretazione dei codisci escape \.
# -d '' stringa vuota come separatore di fine linea, equivale al carattere \0
# IFS=$' \t\n' valore di default di IFS

Se impostassimo IFS=$'\0'

Con xargs:

find . -type f -print0 | xargs -0 -I{} echo {}
