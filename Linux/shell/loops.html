For su files üîó

Following shell script will go though all files stored in /etc directory. The for loop will be abandon when /etc/resolv.conf file found.

#!/usr/bin/env bash
for file in /etc/*; do #  Whitespace-safe but not recursive.
    if [ "${file}" == "/etc/resolv.conf" ]
    then
        countNameservers=$(grep -c nameserver /etc/resolv.conf)
        echo "Total ${countNameservers} nameservers defined in ${file}"
        break
    fi
done

# Make sure globstar is enabled
shopt -s globstar
for file in /etc/**/*; do #  Whitespace-safe and recursive.
    if [ "${file}" == "/etc/resolv.conf" ]
    then
        countNameservers=$(grep -c nameserver /etc/resolv.conf)
        echo "Total ${countNameservers} nameservers defined in ${file}"
        break
    fi
done

Consider using find and xargs üîó

Consider using find and xargs for doing something across a group of files without risking breaking your script because you're operating on too many files (shell wildcard expansion has limits). Using GNU find and xargs:

find . -name '*.foo' -type f -print0 | xargs -0I {} ~/bin/my-script '{}'

will run /bin/my-script on all files (-type f) whose name ends in ".foo" in the current working directory (the dot after the find command) and every directory therein (find is recursive unless you tell it not to be). When you make /bin/my-script all you need to do is describe how to process one file.

find will normally print the list of matching files with a newline. Here we tell find to use NULLs to terminate each list entry (-print0) you can handle pathnames with spaces and other characters that break most scripts. The -0 tells xargs what to use to know where each filename ends. Just don't forget to quote properly in your script (called /bin/my-script here) so the complete pathname is preserved.

I'm also telling xargs where each filename is in the command (-I {} says to use the open/close curly brace as a placeholder for the pathname). This is optional if the last argument to the command is the pathname.

If you have files or directories that have spaces in them, one way to loop over them is with read:

Esempio di come rimuovere gli spazi dai nomi dei file üîó

#!/usr/bin/env bash
for f in *.[Oo][Gg][Gg]; do
newname=`echo $f | tr ' ' '_' | sed -e s/[Oo][Gg][Gg]$/ogg/`
mv "$f" $newname
f=$newname
sox $f tmp.wav
lame -b 128 -q 2 tmp.wav ${f/.ogg/.mp3}
rm -f tmp.wav
done

#!/usr/bin/env bash
# Esempio di comunicazione tra find ed il while di bash
# √® meglio utilizzare il carattere \0 per separare una lista di nomi di files
# perch√® a differenza dello spazio o di altri sicuramente non pu√≤ essere presente nel nome di un file.
# ecco perch√® find possiede l'opzione -print0
# per poter dire a while di aspettarsi una lista separata da \0 si utilizza il parametro -d ''
# Per poter editare un array come "a" non possiamo utilizzare una pipe del tipo (while | find) perch√® bash esegue entrambi i comandi in una subshell,
# e le variabili impostate nella subshell vengono perdute alla sua uscita.
# Uno schema del tipo "while < <(find)" esegue solamente find nella subshell e lo stato di a non viene perduto
# per far parlare la subshell con il while utilizziamo la redirezione aprendo il primo file descriptor >10 libero utilizzabile da bash,
# man bash e cercare il paragrafo "redirection" :
# Each redirection that may be preceded by a file descriptor number may instead be preceded by a word of the form {varname}.  
# In this case, for each redirection operator except >&- and <&-, 
# the shell will allocate a file descriptor greater than or equal to 10 and assign it to varname.  
# If >&- or <&- is preceded by {varname}, the value of varname defines the file  descriptor to  close.  
# If {varname} is supplied, the redirection persists beyond the scope of the command, allowing the shell programmer to manage the file descriptor him‚Äêself.

unset a i
a=()
# -d '' √® equivalente a -d $'\0' 
# while IFS= read -r -u3 -d $'\0' file; do          # in questo esempio utilizziamo il file descriptor 3 sperando che sia disponibile
while IFS= read -r -u"$FD" -d $'\0' file; do        # in questo esempio utilizziamo il file descriptor trovato libero da bash
                                                    # IFS= imposta il carattere del campo di input su nulla, assicurando che non si verifichino divisioni di parole/trim di spazi.
    # a[i++]="$file"                                # or however you want to process each file
    a+=("$file")
# done 3< <(find /tmp -type f -print0 2>/dev/null)  # in questo esempio apriamo il file descriptor 3 sperando che sia disponibile
done {FD}< <(find /tmp -type f -print0 2>/dev/null) # bash cerca un file descriptor disponibile > 10
printf '%s\n' "${a[@]}"                             # esempio di come stampare un array
