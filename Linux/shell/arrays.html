BASH Arrays ðŸ”—

It's hard to quote array values, though, if you don't know ahead of time how many there are! So, using

${foo[*]}

to give all members of the foo array suffers from word-splitting in ksh and bash2 (but not in zsh, by default). In ksh and bash2, though, you can use

"${foo[@]}"

which expands into a quoted list of the members; each member isn't split into separate words.

In csh, $foo[*]:q does the trick.

a convenient way of initializing an entire array is the notation:

array=( element1 element2 ... elementN )

#/usr/bin/env bash

newline='
'
OIFS=$IFS         # Salvo l'input field separator originale
IFS=$newline      # Il nuovo IFS e' un \n cosi gli spazi interni delle parole non daranno luogo a separazioni
files=($(find .)) # La lista dei files viene salvata in un array
IFS=$OIFS        # Ripristino dell'input field separator originale
for i in "${files[@]}"
do
echo $i
done

This produces the correct output:

$ bash t2.sh
.
./file with spaces
./t2.sh
./t1.sh
...

Esempi

#/usr/bin/env bash
array=( zero one two three four five )
# Element 0 1 2 3 4 5

echo ${array[0]} # zero
echo ${array:0} # zero
# Parameter expansion of first element,
#+ starting at position # 0 (1st character).
echo ${array:1} # ero
# Parameter expansion of first element,
#+ starting at position # 1 (2nd character).

echo "--------------"

echo ${#array[0]} # 4
# Length of first element of array.
echo ${#array} # 4
# Length of first element of array.
# (Alternate notation)

echo ${#array[1]} # 3
# Length of second element of array.
# Arrays in Bash have zero-based indexing.

echo ${#array[*]} # 6
# Number of elements in array.
echo ${#array[@]} # 6
# Number of elements in array.
