Bitlocker ðŸ”—

Come montare in automatico mediante /etc/fstab una partizione di un hard disk cifrato con Bitlocker.

Rilevare le caratteristiche UDEV dell'hard disk ðŸ”—

  • https://wiki.archlinux.org/title/udev#Identifying_a_disk_by_its_serial
  • UDEV
$ lsblk -S
NAME HCTL       TYPE VENDOR   MODEL                    REV SERIAL          TRAN
sda  1:0:0:0    disk ATA      Samsung SSD 870 QVO 4TB 1B6Q S5STNF0R301059L sata
$ udevadm info -q all -n /dev/sda1|grep -i serial

Configurazione di un harddisk esterno cifrato con bitlocker mediante UDEV ðŸ”—

Dislocker si trova in AUR, deve essere installato mediante pamac

$ pamac search -a disklocker
$ pamac build dislocker
    sudo mkdir /mnt/bl
    sudo chmod 775 /mnt/bl
    sudo mkdir /mnt/bitlocker
    sudo chmod 775 /mnt/bitlocker

L'hard disk con partizione bitlocker non ha un UUID non si riesce dunque a mapparlo univocamente in /etc/fstab. Utilizzando /dev/sdb1 si riesce a montare automaticamente l'hard disk bitlocker ma non tutti gli altri, per i quali non essendo cifrati il mount automatico tramite /etc/fstab fallisce.

Per risolvere il problema ho utilizzato UDEV intercettando il seriale dell'hard disk bitlocker e facendogli creare il device specifico /dev/bitlocker, che poi ho utilizzato in /etc/fstab al posto di /dev/sdb1.

  1. Regola UDEV per intercettare l'hard disk esterno cifrato bitlocker

    $ sudo bash -c "cat > /etc/udev/rules.d/83-bitlocker.rules" << 'EOF'
  KERNEL=="sd[a-z]1", SUBSYSTEM=="block", ENV{ID_SERIAL_SHORT}=="WD-WXD1E2510M2E", SYMLINK+="bitlocker"
  EOF

  2. Montaggio automatico utilizzando in device creato con UDEV

    $ sudo bash -c "cat >> /etc/fstab" << 'EOF'
/dev/bitlocker                            /mnt/bl           fuse.dislocker user-password=xxxxxxxx,nofail     0       0
/mnt/bl/dislocker-file                    /mnt/bitlocker    auto           nofail                            0       0
EOF

Problema dell'individuazione dell'hard disk cifrato per il mount mediante dislocker in /etc/fstab ðŸ”—

Problema ðŸ”—

L'hard disk con partizione bitlocker non ha un UUID,
non si riesce dunque a mapparlo univocamente in /etc/fstab.
Utilizzando /dev/sdb1 si riesce a montare automaticamente l'hard disk bitlocker ma non tutti gli altri,
nel caso dei quali, che non essendo essi cifrati, il mount automatico tramite /etc/fstab fallisce individuandoli tutti indiscriminatamente con /dev/sdb1.

Soluzione ðŸ”—

Per risolvere il problema ho utilizzato UDEV intercettando il seriale dell'hard disk bitlocker e facendogli creare il device specifico /dev/bitlocker, che poi ho utilizzato in /etc/fstab al posto di /dev/sdb1:

cat > /etc/udev/rules.d/83-bitlocker.rules << EOF
KERNEL=="sd[a-z]1", SUBSYSTEM=="block", ENV{ID_SERIAL_SHORT}=="WD-WXD1E2510M2E", SYMLINK+="bitlocker"
EOF

/etc/fstab ðŸ”—

/dev/bitlocker                 /mnt/bitlocker   fuse.dislocker user-password=xxxxxxxx,nofail 0 0
/mnt/bitlocker/dislocker-file  /media/bitlocker auto nofail 0 0
