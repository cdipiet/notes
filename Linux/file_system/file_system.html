File immutabili non cancellabili ðŸ”—

$ chattr +i myfile.txt

as root will make that file virtually indestructable no matter what user you are. If you want to remove the shield of steel, use -i rather than +i.

$ chattr -i myfile.txt
$ rm myfile.txt

Con chattr +i nemmeno root puo cancellare il file

Creare un file system su file ðŸ”—

# creazione di un file
$ dd if=/dev/zero of=miofilesystem.dd bs=1024 count=10000
# creazione di un file system ext4 nel file
$ mkfs -f -t ext4 miofilesystem.dd
# mount del file formattato come dispositivo di loopback in una directory
$ mkdir test
$ mount -o loop miofilesystem.dd test/
# lettura del file
$ dd if=miofilesystem.dd bs=512 count=2 skip=1|hexdump -C

Difference between Change Time and Modification Time ðŸ”—

The difference between a change and a modification is the difference between altering the label on a package and altering its contents.
If someone says chmod a-w myfile, that is a change;
if someone says echo foo >> myfile, that is a modification.

A change modifies the file's inode; a modification modifies the contents of the file itself.

A file's modification time is also called the timestamp.
