Arch Linux ha deprecato net-tools in favore di iproute2.
il comando ip rimpiazza i tradizionali ifconfig e route

I seguenti comandi di networking di Linux sono deprecati: arp, ifconfig, iptunnel, iwconfig, nameif, netstat, and route.
Questi programmi (eccetto iwconfig) sono inclusi nel pacchetto net-tools che non Ã¨ mantenuto da anni.
Le funzionalitÃ  di molti di questi comandi Ã¨ stata riprodotta e migliorata nella suite iproute2, il comando principale Ã¨ il comando ip. Il codice di iproute2 Ã¨ disponibile presso Kernel.org. La documentazione di iproute2 Ã¨ disponibile presso la Linux Foundation e PolicyRouting.org.

Deprecated command Replacement command(s)
arp ip n (ip neighbor)
ifconfig ip a (ip addr), ip link, ip -s (ip -stats)

iwconfig iw
nameif ip link, ifrename
netstat ss, ip route (for netstat-r), ip -s link (for netstat -i), ip maddr (for netstat-g)

Comandi iproute2 ðŸ”—

$ man ip-link
$ man ip-address
$ man ip-route
$ ip-neighbour
$ ip help       # Display ip commands and arguments
$ ip addr help  # Display address commands and arguments
$ ip link help  # Display link commands and arguments
$ ip neigh help # Display neighbour commands and arguments

$ sudo ip -br -c addr show # find out the IP address assigned to all interfaces and display IP only
$ sudo ip -br -c link show

$ ip link show dev em1           # Display information only for device em1
$ ip link set dev em1 up         # Bring em1 online
$ ip link set dev em1 down       # Bring em1 offline
$ ip link set dev em1 mtu 9000   # Set the MTU on em1 to 9000
$ ip link set dev em1 promisc on # Enable promiscuous mode for em1
$ ip link set mtu 9000 dev eth0  # To change the MTU of the device eth0 to 9000

$ sudo ip addr add 192.168.1.2/24 dev enp5s0
$ ip addr add 192.168.1.1/24 dev em1 # Add address 192.168.1.1 with netmask 24 to device em1
$ ip addr del 192.168.1.1/24 dev em1 # Remove address 192.168.1.1/24 from device em1
$ ip addr add broadcast 172.20.10.255 dev dummy0

$ sudo ip route add 192.168.56.0/24 dev tap0
$ ip route chg default via 192.168.25.110 dev eth1
$ ip route add default via 192.168.1.1 dev em1   # Add a default route (for all addresses) via the local gateway 192.168.1.1 that can be reached on device em1
$ ip route add 192.168.1.0/24 via 192.168.1.1    # Add a route to 192.168.1.0/24 via the gateway at 192.168.1.1
$ ip route add 192.168.1.0/24 dev em1            # Add a route to 192.168.1.0/24 that can be reached on device em1
$ ip route delete 192.168.1.0/24 via 192.168.1.1 # Delete the route for 192.168.1.0/24 via the gateway at 192.168.1.1
$ ip route replace 192.168.1.0/24 dev em1        # Replace the defined route for 192.168.1.0/24 to use device em1
$ ip route get 192.168.1.5                       # Display the route taken for IP 192.168.1.5
# (shows the interface and gateway that would be used to reach a remote host. This command would be especially useful for troubleshooting routing issues on hosts with large routing tables and/or with multiple network interfaces).
$ ip route del default
$ ip route del 192.168.1.0/24 dev eth0

# ARP cache
$ ip n
$ ip neighbor show dev enp5s0
$ sudo ip neigh add 192.168.1.1 lladdr 1:2:3:4:5:6 dev em1 # Add address 192.168.1.1 with MAC 1:2:3:4:5:6 to em1
$ sudo ip neigh del 192.168.1.1 dev em1 # Invalidate the entry for 192.168.1.1 on em1
$ sudo ip neigh replace 192.168.1.1 lladdr 1:2:3:4:5:6 dev em1 # Replace the entry for address 192.168.1.1 to use MAC 1:2:3:4:5:6 on em1
$ sudo ip -s -s n f 192.168.1.5         # Flush or f command flushes neighbour/arp tables for 192.168.1.5
$ sudo ip -s -s neigh flush all # pulisce tutta la cache ARP, il primo -s Ã¨ per avere un output verboso
