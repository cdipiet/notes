Esecuzione come utente non root
$ nmap 127.0.0.1
# By default, when running as a non-root account, Nmap does a full TCP connect scan, 
# completing the TCP three-way handshake for each open TCP port on the target.
# A   -->SYN   B
# A SYN-ACK<-- B
# A   -->ACK   B
# con --reason si vede il motivo delle conclusioni di nmap
$ nmap --reason 127.0.0.1
# When you run Nmap as a root user, the default scan type changes. As non-root, 
# Nmap uses a TCP connect() to determine if a port is open or closed. 
# As root, Nmap uses a half-open or TCP SYN scan (a TCP SYN is sent to start a connection, but the connection is never completed). 
$ sudo nmap --reason -p 1-1024 127.0.0.1
# you can specify -p 1-65535, or use the Nmap shorthand -p-. Scan all 65535 ports on the Slingshot Linux host, as shown here:
$ sudo nmap --reason -p- 127.0.0.1

Subnet scan - trovare gli host attivi nella sottorete ðŸ”—

Per trovare velocemente gli host attivi non facciamo un port scan, lo faremo in seguito sui singoli host di interesse trovati.

# -sL: List Scan - simply list targets to scan
$ nmap -sL 192.168.1.2/24
# Nelle versioni precedenti di Nmap, l'opzione -sn era chiamata -sP
#  -sn (No port scan) Questa opzione indica a Nmap di non effettuare un port scan dopo un host discovery 
$ nmap -sn 192.168.1.2/24
$ nmap -sP 192.168.1.2/24
