Control groups (or cgroups as they are commonly known) are a feature provided by the Linux kernel to manage, restrict, and audit groups of processes.

  • For cgroup v2, the output is cgroup2fs.
  • For cgroup v1, the output is tmpfs.

Enable cgroup v2 ðŸ”—

https://sleeplessbeastie.eu/2021/09/10/how-to-enable-control-group-v2/

Se abbiamo abilitato cgroups v1 risulterÃ :

$ cat /sys/fs/cgroup/cgroup.controllers
cat: /sys/fs/cgroup/cgroup.controllers: No such file or directory
$ stat -c %T -f /sys/fs/cgroup
tmpfs

Per abilitare cgroup v2:

# To enable Control Group v2 alter boot options to append systemd.unified_cgroup_hierarchy=1 parameter.
$ cat /etc/default/grub | grep GRUB_CMDLINE_LINUX=
GRUB_CMDLINE_LINUX=""

$ sudo sed -i /GRUB_CMDLINE_LINUX=/'c\GRUB_CMDLINE_LINUX="systemd.unified_cgroup_hierarchy=1 cgroup_no_v1=all"' /etc/default/grub
# In alternativa:
$ sudo sed -i -e 's/^GRUB_CMDLINE_LINUX=""/GRUB_CMDLINE_LINUX="systemd.unified_cgroup_hierarchy=1 cgroup_no_v1=all"/' /etc/default/grub

$ sudo update-grub
$ sudo reboot
# Verifica che sia abilitato v2
$ stat -fc %T /sys/fs/cgroup
cgroup2fs
$ cat /sys/fs/cgroup/cgroup.controllers
cpuset cpu io memory hugetlb pids rdma misc

You can tell systemd to use cgroupv2 via kernel cmdline parameter:

systemd.unified_cgroup_hierarchy=1

It might also be needed to explictly disable hybrid cgroupv1 support to avoid problems using: systemd.legacy_systemd_cgroup_controller=0 Or completely disable cgroupv1 in the kernel with: cgroup_no_v1=all

You can view your host systemd environment with:

$ systemctl show -all | grep default-hierarchy
$ ps -p 1 -o comm=
