  <name>default</name>

Share samba nell'host ðŸ”—

Per condividere una directory tra host e guest windows 10
bisogna attivare samba sull'host, configurare una directory condivisa (es. "Share"),
ed accedere alla share nell'host da windows 10 mediante percorso samba \\192.168.1.2\Share Per prima cosa impostiamo un ip statico per Windows 10 in modo da facilitare la configurazione di samba nell'host, impostiamo l'ip statico senza toccare windows 10, configurando la rete default di libvirt.

impostare un ip statico per windows 10 da virt-manager utilizzando la configurazione del dhcp per la rete default: ðŸ”—

configuriamo il DHCP della rete default di libvirt in modo che al MAC address della macchina windows 10 corrisponda sempre lo stesso indirizzo IP.

ip statico per win10: 192.168.122.10

$ sudo virsh  dumpxml  win10 | grep 'mac address'
      <mac address='52:54:00:f8:e7:60'/>

$ sudo virsh net-list
$ sudo virsh net-edit default

# Find the <dhcp> section, restrict the dynamic range and add host entries for your VMs

    <dhcp>
    <range start='192.168.122.100' end='192.168.122.254'/>
    <host mac='52:54:00:f8:e7:60' name='win10' ip='192.168.122.10'/>
    </dhcp>

$ sudo virsh net-destroy default
$ sudo virsh net-autostart default
$ sudo virsh net-start default  
$ sudo virsh net-dumpxml default

# Then, reboot your VM

Configurazione di una directory condivisa Samba nell'host ðŸ”—

smb.conf nell'host configurato con l'ip statico della vm windows 10.

Assegneremo l'ip statico 192.168.122.10 alla vm windows 10,
configuriamo la protezione della share in modo da accettare le connessioni alla share anche dall'ip 192.168.122.10:

$ sudo vim /etc/samba/smb.conf

    # A private directory, usable only by $USER. Note that $USER requires write
    # access to the directory.
    [Share]
    comment = $USER's Service
    path = /home/$USER/Share
    valid users = $USER
    hosts allow = 192.168.1.4 127.0.0.1 localhost 192.168.122.10
    public = no
    writable = yes
    printable = no

$ sudo testparm
# Rilettura del smb.conf
$ sudo systemctl restart smb.service
$ sudo smbcontrol smbd reload-config
