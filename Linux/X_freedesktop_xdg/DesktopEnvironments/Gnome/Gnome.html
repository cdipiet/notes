Autostart with Systemd service ðŸ”—

We can also get XScreenSaver autostart at boot time with Systemd First, execute the following command to create a directory for storing Systemd service file of individual users.

mkdir -p ~/.config/systemd/user/
cd ~/.config/systemd/user/

Then create xscreensaver.servicefile with nano text editor.

nano ~/.config/systemd/user/xscreensaver.service

Put the following text into the file.

[Unit]
Description=XScreenSaver
[Service]
ExecStart=/usr/bin/xscreensaver -nosplash
[Install]
WantedBy=default.target

Press CTRL+O to save the file. Press CTRL+X to exit out of nano text editor. Finally, enable this service so that XScreenSaver will be automatically started at boot time. systemctl --user enable xscreensaver Created symlink /home/cdp/.config/systemd/user/default.target.wants/xscreensaver.service â†’ /home/cdp/.config/systemd/user/xscreensaver.service.

Per cambiare la scala bisogna ora editare il file ~/.config/monitors.xml:

swap-res.sh

#!/bin/bash
File=~/.config/monitors.xml
if grep -q "<scale>1" "$File"; then
    sed -i -e '/<scale>1/ s/1/2/' "$File"
else
    sed -i -e '/<scale>2/ s/2/1/' "$File"
fi

The monitors.xml file stores the monitors settings as configure by Gnome System Settings. It can store configurations for multiple monitor setups and the user's personal choice of screen resolution. Gnome System Settings uses the XRANDR extension to reconfigure the display(s) on the fly.
